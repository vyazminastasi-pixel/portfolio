Case 01 — Redesign

Crypto Deposit
Redesign

The original deposit screen offered no currency choice, no context, and presented the minimum deposit as a large barrier. Users dropped off before completing the transaction.

The redesign simplified the flow, added multi-currency selection, embedded a retention mechanic, and restructured the layout for clarity on mobile.

Case 03 — New Feature

P2P Payment
Matching System

A new internal feature that matches incoming deposit requests with pending withdrawal requests — allowing users to transfer funds directly between each other, bypassing third-party processors and their fees.

The system shows users only the deposit amounts that match active withdrawal queues. If no withdrawals are pending, the method is hidden or shows "temporarily unavailable."
